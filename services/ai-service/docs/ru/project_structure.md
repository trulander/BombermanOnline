# Структура проекта

Ниже представлена структура каталогов и файлов сервиса с описанием их назначения.

```
services/ai-service/
├── app/
│   ├── __init__.py
│   ├── main.py                   # Точка входа FastAPI приложения, инициализация сервиса.
│   ├── config.py                 # Конфигурационные настройки, управляемые переменными окружения.
│   ├── nats_controller.py        # Логика обработки сообщений из NATS (подписки и публикации).
│   ├── agents/                   # Реализации AI-агентов.
│   │   ├── base_agent.py         # Абстрактный базовый класс для всех агентов.
│   │   ├── bomberman_ai_agent.py # Агент, управляющий игроком-AI.
│   │   └── enemy_ai_agent.py     # Агент, управляющий врагами.
│   ├── entities/                 # Pydantic-модели для представления данных.
│   │   └── game_state_representation.py # Модель для десериализации состояния игры.
│   ├── environment/              # Среда Gymnasium для обучения AI.
│   │   └── bomberman_env.py      # Кастомная среда обучения, симулирующая игру Bomberman.
│   ├── registry/                 # Реестры для управления объектами в памяти.
│   │   └── ai_agent_registry.py  # Реестр для хранения и управления активными AI-агентами.
│   ├── repositories/             # Репозитории для взаимодействия с внешними системами.
│   │   ├── nats_repository.py    # Абстракция для работы с NATS.
│   │   └── redis_repository.py   # Абстракция для работы с Redis.
│   └── training/                 # Логика для тренировки моделей.
│       ├── model_manager.py      # Управление сохранением и загрузкой моделей.
│       └── trainer.py            # Оркестратор процесса обучения моделей.
├── Dockerfile                    # Определение Docker-образа для сервиса.
├── pyproject.toml                # Определение зависимостей и конфигурация проекта.
├── README.md                     # Документация на английском языке.
└── README_RU.md                  # Основной файл документации на русском языке.
```
